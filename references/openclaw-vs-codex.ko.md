# OpenClaw vs Codex 스킬 크리에이터

> 번들 포함된 Codex 스킬 크리에이터(`/app/skills/skill-creator/`)와 새로운 OpenClaw 네이티브 스킬 크리에이터의 차이점을 문서화합니다. 새로운 OpenClaw 스킬을 생성하거나 기존 Codex 스킬을 포팅할 때 이 참고 자료를 사용하세요.

## 아키텍처 차이

| | Codex (CLI 코딩 에이전트) | OpenClaw (멀티 에이전트) |
|---|---|---|
| 실행 | 단일 CLI 세션 | 메인 + sub-agent (spawn/fork) |
| 컨텍스트 | 모든 컨텍스트가 세션 내에 존재 | sub-agent는 **빈** 컨텍스트로 시작 |
| 스킬 로딩 | description → body → scripts (순차적) | description(항상) → body(트리거 시) → references(요청 시) |
| 스킬 저장소 | 단일 위치 | workspace(재시작 간 유지) vs bundled(업데이트 시 초기화) |
| 프롬프트 주입 | 불필요 (단일 컨텍스트) | **필수** (sub-agent에 명시적 주입) |
| 패키징 | `package_skill.py`이 `.skill` 파일 생성 | 패키징 불필요 — 폴더 구조를 직접 인식 |

## 5가지 핵심 차이

### 1. Sub-Agent 컨텍스트 주입

**Codex**: 모든 작업이 하나의 세션에서 이루어지므로 컨텍스트 공유가 자연스럽습니다.

**OpenClaw**: sub-agent를 spawn하면 컨텍스트가 **비어 있습니다**. 메인 에이전트가 Phase 0에서 수집한 정보를 sub-agent 프롬프트에 명시적으로 주입해야 합니다.

Codex 스킬에는 이 개념이 없습니다. OpenClaw에서는 가장 중요한 패턴입니다.

```
Codex:   "이 텍스트를 요약해줘" → OK (컨텍스트가 이미 존재)
OpenClaw: "이 텍스트를 요약해줘" → FAIL (sub-agent에 컨텍스트가 없음)

OpenClaw 올바른 접근:
"이 텍스트는 'OOO 사용자 매뉴얼'의 섹션 1~5입니다.
전체 문서는 5개 파트에 27개 섹션이 있습니다. 이 청크는 '새로운 설계 방법론'을 다룹니다.
다음을 보존하세요: [비유], [인용구], [사례 연구]..."
```

### 2. Style Guide 우선

**Codex**: "간결함이 핵심" — 컨텍스트 윈도우 절약에 집중합니다. 에이전트가 이미 충분히 똑똑하다고 가정합니다.

**OpenClaw**: Style Guide가 없으면 sub-agent는 "텍스트 압축기"가 됩니다. **판단 기준**이 필수입니다.

Codex 스킬은 형식(구조, 길이)에 집중하지만, OpenClaw 스킬은 판단 기준(무엇을 보존할지, 어떤 어조로, 어떤 관점에서)도 포함해야 합니다.

### 3. Phase 0 (분석 단계) 필수

**Codex**: 에이전트가 작업하면서 자연스럽게 분석합니다.

**OpenClaw**: 메인 에이전트가 Phase 0에서 분석한 뒤, 결과를 sub-agent 프롬프트에 주입하는 파이프라인이 필요합니다.

Phase 0이 없으면:
- 각 청크가 독립적으로 처리됩니다 (전체 흐름이 상실됨)
- 청크 간 중복 콘텐츠가 나타납니다
- 저자의 목소리(비유, 인용구)가 누락됩니다

### 4. 순차적 파이프라인

**Codex**: 여러 출력을 병렬로 생성할 수 있으며, 에이전트가 자연스럽게 병합합니다.

**OpenClaw**: 복잡한 출력은 단순에서 복잡으로 순차적으로 확장해야 합니다.

예: 요약 깊이 1x + 5x
- Codex: 병렬 생성 가능 (에이전트가 병합)
- OpenClaw: 1x → 5x 순차적 (1x 구조가 5x의 뼈대 역할)

### 5. 패키징 불필요

**Codex**: `package_skill.py`이 배포용 `.skill` 파일(사실상 zip)을 생성합니다.

**OpenClaw**: 폴더 구조를 직접 인식합니다. 스킬이 작동하는 데 `SKILL.md`만 있으면 됩니다.

패키징 스크립트, 검증 스크립트, init 스크립트가 필요하지 않습니다.

## 포팅 체크리스트: Codex 스킬 → OpenClaw 스킬

Codex 스킬을 OpenClaw로 포팅할 때:

- [ ] **sub-agent 필요 여부 확인** — 스킬이 청킹이나 병렬 처리를 수행하는 경우, Phase 0 + 컨텍스트 주입 패턴이 있는지 확인
- [ ] **Style Guide 있는지?** — 없으면 추가하세요. 형식 스펙만으로는 sub-agent가 고품질 출력을 생성하기 부족합니다
- [ ] **Phase 0 추가** — 작업 시작 전 분석 단계가 있는지?
- [ ] **`[ ]` 플레이스홀더가 있는 프롬프트 템플릿** — 플레이스홀더가 Phase 0 결과로 채워지는지 확인
- [ ] **병합 지침** — 다중 청크 출력의 단순 연결을 명시적으로 방지
- [ ] **`package_skill.py` 참조 제거** — OpenClaw에서 불필요
- [ ] **`init_skill.py` 참조 제거** — OpenClaw에서 불필요
- [ ] **description에 트리거 키워드** — 영어 및 현지화된 키워드 모두 포함
- [ ] **"사용 시점"을 body에서 description으로 이동** — OpenClaw는 트리거 후에만 body를 로드합니다

## Codex에서 유지할 패턴

다음은 두 환경 모두에서 잘 작동하는 좋은 패턴입니다:

1. **Progressive Disclosure** — SKILL.md를 500줄 이하로 유지하고, 세부 사항은 references/로 이동
2. **"간결함이 핵심" 원칙** — 에이전트가 이미 아는 것을 다시 설명하지 마세요
3. **자유도 조절** — 작업의 취약성(높음/중간/낮음)에 따라 구체성을 조정
4. **단일 레벨 참조 깊이** — references/ 내에서 다른 references/를 참조하지 마세요
5. **사용 후 개선** — 스킬을 사용하고, 실패를 관찰하고, 수정하고, 반복
